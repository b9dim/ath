# تطبيق التأخير (at)

تطبيق ويب يعتمد على Next.js ويحاكي تجربة تطبيقات iOS 18/26 لحساب التأخير اليومي في الوصول إلى الدوام مقارنة بالوقت الرسمي 07:15 صباحًا، مع حفظ السجل محليًا.

## فكرة المشروع

يتيح التطبيق إدخال وقت الحضور الفعلي، ثم يقارن الوقت بالموعد الرسمي للوصول (07:15 صباحًا) ليحدد مقدار التأخير بالدقائق، مع واجهة عربية من اليمين لليسار وتجربة بصرية تشبه تطبيقات iPhone الحديثة.

## طريقة الحساب

- الوقت الرسمي للوصول ثابت على 07:15 صباحًا.
- عند إدخال وقت الحضور:
  - إذا كان الوقت ≤ 07:15: لا يوجد تأخير ويعرض التطبيق رسالة نجاح.
  - إذا كان الوقت > 07:15: يحسب الفرق بالدقائق ويعرض التأخير.
  - إذا بلغ التأخير 420 دقيقة أو أكثر (ما يعادل يوم عمل كامل): يتم التنبيه بذلك بشكل واضح.
- يتم تسجيل كل عملية حساب في سجل يحتوي على التاريخ، ووقت الحضور، وعدد دقائق التأخير.

## حفظ البيانات (LocalStorage)

- يتم حفظ السجل في المتصفح فقط عبر المفتاح `attendance_log`.
- يخزن كل سجل البيانات التالية:
  - `date`: تاريخ اليوم بصيغة `YYYY-MM-DD`.
  - `arrivalTime`: وقت الحضور المدخل.
  - `delayMinutes`: مقدار التأخير بالدقائق (يمكن أن يكون صفرًا).
  - `createdAt`: الطابع الزمني للحفظ.
- لا يتم إرسال أي بيانات لخادم خارجي؛ كل المعلومات تبقى على جهاز المستخدم.
- يمكن حذف السجل بالكامل من صفحة "سجل التأخير" عبر ورقة تأكيد (Bottom Sheet) على نمط iOS.

## المتطلبات

- Node.js 18 أو أحدث.
- npm أو pnpm أو Yarn (يفضل npm).

## التشغيل محليًا

```bash
# تثبيت الاعتمادات
npm install

# تشغيل بيئة التطوير
npm run dev
```

بعد التشغيل، افتح المتصفح على العنوان `http://localhost:3000`.

## البناء والإنتاج

```bash
# إنشاء نسخة الإنتاج
npm run build

# تشغيل النسخة المبنية
npm start
```

## النشر على Vercel

1. أنشئ حسابًا على [Vercel](https://vercel.com/).
2. اربط مستودع المشروع بحسابك في Vercel.
3. عند طلب إعدادات البناء:
   - أمر البناء: `npm run build`
   - مسار الإخراج: `.next`
   - نسخة Node المقترحة: 18
4. اضغط Deploy وستتولى Vercel نشر التطبيق تلقائيًا.

## النشر على Render

1. أنشئ حسابًا على [Render](https://render.com/).
2. أنشئ خدمة ويب جديدة (Web Service) واختر مستودع المشروع.
3. اضبط الإعدادات التالية:
   - بيئة التنفيذ: `Node`
   - أمر التثبيت: `npm install`
   - أمر البناء: `npm run build`
   - أمر التشغيل: `npm start`
4. أنشئ الخدمة وانتظر حتى تكتمل عملية البناء والنشر.

## التقنية المستخدمة

- **Next.js (App Router)** لتطبيق React حديث بدون خادم.
- **Tailwind CSS** لتخصيص النمط وملاءمة تصميم iOS.
- **Framer Motion** للانتقالات والحركات السلسة.
- **LocalStorage** لحفظ السجل محليًا بدون قاعدة بيانات خارجية.
